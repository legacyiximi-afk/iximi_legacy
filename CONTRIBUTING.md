# Guía de Contribución para IXIMI Legacy

¡Gracias por tu interés en contribuir a IXIMI Legacy! Estamos muy contentos de tenerte aquí. Cada contribución, sin importar cuán grande o pequeña sea, es valiosa para nosotros.

## Cómo Puedes Contribuir

Hay muchas maneras de contribuir a nuestro proyecto:

*   **Reportando Bugs:** Si encuentras un error, por favor, [abre un issue](https://github.com/legacyiximi-afk/iximi_legacy/issues/new?template=bug_report.md) y descríbelo con el mayor detalle posible.
*   **Sugiriendo Mejoras:** ¿Tienes una idea para una nueva característica o una mejora? [Abre un issue](https://github.com/legacyiximi-afk/iximi_legacy/issues/new?template=feature_request.md) y cuéntanos sobre ella.
*   **Escribiendo Código:** Si quieres escribir código, por favor, revisa los [issues abiertos](https://github.com/legacyiximi-afk/iximi_legacy/issues) y elige uno para trabajar. También puedes proponer tus propios cambios.
*   **Mejorando la Documentación:** La buena documentación es clave. Si ves algo que se puede mejorar, no dudes en enviar un pull request.

## Proceso de Desarrollo

1.  **Haz un Fork:** Haz un fork de este repositorio a tu propia cuenta de GitHub.
2.  **Clona tu Fork:** Clona tu fork a tu máquina local.
3.  **Crea una Rama:** Crea una nueva rama para tus cambios. Usa un nombre descriptivo, como `feat/nueva-caracteristica` o `fix/error-de-autenticacion`.
4.  **Haz tus Cambios:** Haz los cambios que quieras en tu rama local.
5.  **Prueba tus Cambios:** Asegúrate de que tus cambios no rompan nada. Ejecuta las pruebas existentes y, si es posible, añade nuevas pruebas para tus cambios.
6.  **Haz Commit:** Haz commit de tus cambios con un mensaje claro y descriptivo.
7.  **Haz Push:** Haz push de tus cambios a tu fork en GitHub.
8.  **Abre un Pull Request:** Abre un pull request desde tu fork a nuestro repositorio. Asegúrate de llenar la plantilla de pull request con toda la información necesaria.

## Estándares de Código

*   **Estilo de Código:** Usamos Prettier y ESLint para mantener un estilo de código consistente. Por favor, asegúrate de que tu código sigue nuestras reglas de estilo.
*   **Mensajes de Commit:** Usamos [Conventional Commits](https://www.conventionalcommits.org/) para nuestros mensajes de commit. Esto nos ayuda a generar automáticamente el changelog y a mantener un historial de cambios limpio.

## Código de Conducta

Al participar en este proyecto, te comprometes a seguir nuestro [Código de Conducta](CODE_OF_CONDUCT.md). Queremos que nuestra comunidad sea un lugar seguro y acogedor para todos.

## ¿Preguntas?

Si tienes alguna pregunta, no dudes en [abrir un issue](https://github.com/legacyiximi-afk/iximi_legacy/issues/new) o unirte a nuestra [comunidad en Discord](https://discord.gg/iximi-legacy).
